Лабораторная работа №1 — Кодирование информации

Создать приложение с графическим интерфейсом в любой визуйльной среде разработки, выполняющее следующие действия:

  • Пользователь вводит текст. Количество символов не ограничивается. Допускаются буквы кириллицы и прочие видимые знаки.
  • Выполняется кодирование символов в кодировку, заданную индивидуальным вариантом задания.
  • Закодированный текст выводится в двоичном виде на экран (для удобства блоки цифр разделяются пробелами и т.п.).
  • Возможно и обратное преобразование — декодирование.
Предусмотреть возможность сохранения кодированного текста в бинарные (как в памяти) и в текстовые (как на экране) файлы.

Варианты (mod 4)

  1. Windows-1251
  2. KOI8-R
  3. ISO 8859-5
  4. ASCII/CP-866

Лабораторная работа №2 — Передача закодированной информации. UDP

Создать приложение с графическим интерфейсом в любой визуйльной среде разработки, выполняющее следующие действия:

  • Пользователь вводит текст. Количество символов не ограничивается. Должны быть реализованы буквы кириллицы верхнего и нижнего регистра и знаки пробела.
  • Выполняется кодирование символов в кодировку, которое было реализовано в лабораторной работе №1.
  • Закодированный текст выводится в кодированном и двоичном виде на экран (для удобства блоки цифр разделяются пробелами и т.п.).
  • Происходит передача кодированного текста на другой компьютер:
    - для этого требуется реализовать в программе прием и передачу данных используя протокол UDP
  • На другом компьютере, использую эту же программу для первого компьютера, происходит вывод полученных сообщений в двоичном, кодированном и декодированном виде (UTF).
  • При каждом запуске программы создавать новый чистый текстовы файл (имя файла: dd.MM.YYYY_hh-mm-ss.txt), в который записывать все входящие сообщения в закодированном виде.

Лабораторная работа №3 — Передача закодированной информации. TCP

Скопировать код лабораторной работы №2 в новый проект. Изменить протокол UDP на TCP. Для сдачи должны быть реализованы следующие возможности программы:

  • Пользователь вводит текст. Количество символов не ограничивается. Должны быть реализованы буквы кириллицы верхнего и нижнего регистра и знаки пробела.
  • Выполняется кодирование символов в кодировку, которое было реализовано в лабораторной работе №1.
  • Закодированный текст выводится в кодированном и двоичном виде на экран (для удобства блоки цифр разделяются пробелами и т.п.).
  • Происходит передача кодированного текста на другой компьютер:
    - для этого требуется реализовать сервер и клиент, которые обеспечат прием и передачу данных используя протокол TCP
  • На другом компьютере, использую клиент для первого компьютера, происходит вывод полученных сообщений в двоичном, кодированном и декодированном виде (UTF).
  • В программе сервере происходит вывод полученных сообщений от обоих клиентов в двоичном, кодированном и декодированном виде (UTF), с указанием от какого клиента они пришли.
  • При каждом запуске клиента создавать новый чистый текстовы файл (имя файла: dd.MM.YYYY_hh-mm-ss.txt), в который записывать все входящие сообщения в закодированном виде.
  • При каждом запуске сервера создавать новый чистый текстовы файл (имя файла: dd.MM.YYYY_hh-mm-ss.txt), в который записывать все входящие сообщения с двух клиентов в закодированном виде.

Лабораторная работа №4 — Доработка протокола UDP

Скопировать код лабораторной работы №2 в новый проект. Доработать программу следующим образом:

  1. Реализовать класс UDPMessage: 
    class UDPMessage
{
	bool IsCheck;
	int Length;
	byte[] Message;
}
  2. Реализовать меню выбора функций программы.
  3. ВСЕ сообщения должны отправляться в виде экземпляра класса UDPMessage в закодированном виде. Поле Message содержит текст в вашей кодировке.
  4. Реализовать функцию проверки соединения. Для этого требуется отправлять экземпляр класса UDPMessage со значением поля IsCheck = true.
  5. Другой экземпляр программы должен в ответ на такое сообщение отправить ответ (используя экземпляр класса UDPMessage) с текстом "Проверка пройдена".
  6. Если значение поля IsCheck = false, то программа должна выводить сообщение и сохранять его в файл, как это описано в лабораторной работе №2
  7. При приёме сообщений использовать для проверки целостности поле Length. Данное поле заполняется на этапе отправки и оно должно содержать длину поля Message.
  8. На принимающем компьютере сверять значение этого поля со значением длины поля сообщения: Length == Message.Length
  9. Если они равны, то программа должна выводить сообщение и сохранять его в файл, как это описано в лабораторной работе №2
  10. Если они не равны, вывести на экран принимающего компьютера текст "Сообщение повреждено!"
